# 香港中文問答規則設計

## 背景

目前 `AGENTS.md` 已定義程式語言、開發流程、i18n 與文件要求，但未有一條明確規則要求代理與使用者互動時必須使用香港中文。因此，即使 repo 已偏向 `zh-HK`，代理回覆語言仍可能不一致。

## 目標

- 在 `AGENTS.md` 明確加入使用者互動語言規則
- 要求所有問答、解釋、一般回覆預設使用香港中文（`zh-HK`）
- 將對話語言規則與現有 i18n 系統字串要求清晰區分

## 非目標

- 不修改應用程式執行時的 i18n resource 結構
- 不更動程式碼、測試或執行邏輯
- 不重新整理整份 `AGENTS.md` 的章節結構

## 方案選擇

### 方案 A：新增全域語言規則段落（採用）

在 `AGENTS.md` 前段加入簡短的 `Communication Language` 或同類章節，直接寫明所有對使用者的問答、解釋與回覆必須使用香港中文。

優點：

- 規則最清晰直接
- 容易被代理優先讀到
- 不會與產品 i18n 定義混淆

缺點：

- 章節數量略增

### 方案 B：寫入現有 i18n 段落

將對話語言規則放入 `## Internationalization (i18n) Requirements`。

優點：

- 內容集中

缺點：

- 容易被理解成只限系統字串或產品介面
- 不夠明確覆蓋代理對話

## 設計

### 插入位置

在 `AGENTS.md` 前半部加入新段落，位置應早於較細節的工程實作規範，令代理在解讀整體行為時優先見到。

### 規則內容

新增規則將明確表述：

- 所有與使用者的問答、解釋、說明及一般回覆必須使用香港中文
- 預設語言 locale 為 `zh-HK`
- 如使用者明確要求其他語言，才按使用者要求切換

### 與現有內容關係

- 新規則負責代理對話語言
- 現有 i18n 段落繼續負責系統字串、資源檔與 locale 配置
- `Additional Notes` 內關於香港繁體中文的描述可保留，作為文件與註解層面的補充

## 驗收準則

- `AGENTS.md` 出現一條清晰、不可誤解的香港中文問答規則
- 規則可直接套用於代理所有使用者回覆
- 新規則與現有 i18n 要求沒有衝突

## 實作影響

- 只涉及 `AGENTS.md` 一個文件
- 不需要資料庫、API 或測試變更

## 備註

按目前工作規則，未獲使用者明確要求前不會建立 git commit，因此本設計文件只會寫入工作目錄，不會自動提交。
